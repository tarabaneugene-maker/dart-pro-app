```
┌──────────────────────────────────────┐
│ ← | 501 | 1с | 3л | DO      | ⚡   │  ← Строка статуса
├──────────────────────────────────────┤
│                                      │
│  ┌── ВЕРХНЯЯ ПОЛОВИНА ЭКРАНА ────┐  │
│  │  🎯 ИГРОК 1 — ОСТАЛОСЬ 86    │  │  ← Плашка активного игрока
│  │     Закрытие: T20-T16-DBull   │  │      Если ≤170 — checkout варианты
│  ├────────────┬──────────────────┤  │
│  │ Игрок 1    │  2с  Игрок 2     │  │  ← Левая: Имя | счётчик легов
│  │   ср:52.3  │    ср:48.1       │  │      Правая: счётчик легов | Имя
│  │   501      │         301      │  │      Тап по имени — тоггл average
│  │         3  │           6      │  │      В правом углу — дротиков в леге
│  │   ← 100   │       ← 81       │  │  ← Последний подход (сумма/дротики)

│  │  или [45][60][81][85][100][140]│  │  ← Режим А: быстрые суммы

├──────────────────────────────────────┤
│  ┌── НИЖНЯЯ ПОЛОВИНА ЭКРАНА ────┐  │
```

### Фаза 2: Онлайн (в разработке)

#### Архитектура

```
КЛИЕНТ (Flutter)                    СЕРВЕР (Dart WebSocket)
  │                                      │
  │─── WS connect (JWT) ────────────────►│
  │─── create_room / join_room ─────────►│
  │◄── room_created / player_joined ─────│
  │◄── game_started ─────────────────────│
  │─── throw (score) ───────────────────►│
  │◄── throw_result (sync) ──────────────│
  │─── ping (каждые 15с) ───────────────►│
  │◄── pong ─────────────────────────────│
```

#### Защита от зависаний

| Механизм | Описание |
|----------|----------|
| **Heartbeat (ping/pong)** | Каждые 15с. 3 пропуска = таймаут (45с) |
| **Reconnection window** | Переподключение до 60с. Сервер хранит комнату 120с |
| **State reconciliation** | После переподключения — сверка состояния с сервером |
| **Turn timeout** | 120с без хода → opentime. Ещё 60с → победа соперника |
| **Game snapshot** | После каждого лега — сохранение в БД |
